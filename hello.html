Os comandos principas para connector a um reparatorio GITHUB :

  1. git clone "link do reparatorio" : copiar do Reparatorio GITHUB para o computador
  2. touch nomedoficheiro.html
  3. git add nomedoficheiro
  4. git commit -m "observação do que se fez"
  5. git status : nos permite ver onde nos situamos no repatorio
  6. git push : publicar as alterções do computador para o GITHUB
  7. git commit -am "observação da alteração que se fez" : Serve para commit todas as alterações
  8. git pull : para copiar as mudanças feitas no GITHUB para o computador

Resolver conflitos : Marge Conflicts

Esses conflitos aparecem quanod duas pessoas estão a fazer mudanças simultaneamente no mesmo ficheiro e na mesma linha

Outros comandos de GIT

  • git log : Serve para verificar todas as mudanças no seu codigo num dado reparatorio
  • git reset --hard <\ nº do commit> : Serve para anular as mudanças faitas no ficheiro de um reparatorio
  • git reset --hard origin/master : Serve para recuparar o ficheiro orignal

Branching

Permite criar uma parte no reparatorio, permitindo de traballhar em diferentes parte do reparatorio no mesmo momento

  • git branch : para verificar em qual "branch"
  • git checkout -b nomedabrance : permite criar uma nova Branch
